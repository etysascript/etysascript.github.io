Introduction

Welcome to the Etysa reference documentation! This is better for people who already have some coding knowledge with any programming language as learning won't taken at all a while.

If you know non, a friend good at programming can assist but a book will be released that will catalyse everything. Plus tutorials too. An interactive console will soon come where you can learn as you test your coding live on your browser.

Just some sad news is that etysa is very buggy beyond beta version. So expect error but they will be sorted.

Installation and Builds

Distibution is not direct for as you may have to compile the source code for your system. Good etysa is written in Dlang, a language capable of setting on any system so just download and using following commands.

Source is at github assets, Make sure a Dlang compiler is installed on your pc, DMD is recommended.

$ git clone https://github.com/etysascript/etysa.git
$ cd etysa
$ make build
$ ./etysa -v

Just some sad news is that etysa is very buggy beyond beta version. So expect errors but they will be sorted, but at this moment you have the executable ready to run.

ETYSA SYNTAX

Syntax

Variables

Variables should have a name without spaces and data they hold. There is no need for decorators, and they hold any type of data that changes during runtime.

let and var keywords are just mathematic expressions to detail the variable.

Datatypes

Etysa has few datatypes but you may create custom ones using classes.

Quick custom data using enums.

Functions

Functions create reuseable code blocks, they can be given parameters that require arguments during function calling. They have there own scope as variables created in theme won't reflect outside.

Functions can nest other blocks including functions too, using the same function name overwrites the current one in the scope.

if elif else statement

Conditional execution is achieved by these blocks, elifs can be any number as long as between if plus else statements.

try except statement

Escapes errors in a program, returns the error messages and the program continues.

While loop

Loop conditional must be fulfilled to continue execution or stops but can be broken out if target reached.

For loop

Two for loops exists, one iters the given iterable data and the other depends on a conditions.
